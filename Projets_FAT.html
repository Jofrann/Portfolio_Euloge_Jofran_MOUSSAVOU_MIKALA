Développement Fiche d'Assignation de Tâches

Contexte

Dans le cadre de la gestion d'équipes et de projets, une fiche d'assignation de tâches est nécessaire pour une meilleure organisation. L'application a pour objectif de faciliter l'assignation, le suivi et la gestion des tâches entre utilisateurs. Elle permet aux gestionnaires de projets de mieux gérer l'attribution des tâches et à chaque membre de suivre l'avancement de celles-ci.

Le projet a été conçu pour répondre à des besoins croissants d'efficacité dans le suivi des tâches entre plusieurs utilisateurs au sein d'une organisation.

Besoins

  • Créer un système de gestion des tâches avec assignation à différents utilisateurs.

  • Permettre la visualisation et mise à jour des tâches par les utilisateurs et gestionnaires.

  • Mettre en place un suivi de l'avancement des tâches.

  • Intégration avec une base de données MySQL pour stocker les informations des tâches et des utilisateurs.

  • Simplifier l'interface pour une utilisation facile et rapide.

Mise en œuvre

Le développement a été divisé en plusieurs étapes clés :

  • Conception de la base de données : J'ai créé une base de données relationnelle avec MySQL, où les informations des utilisateurs, des tâches et leur progression étaient stockées.

  • Développement du back-end : J'ai utilisé PHP pour gérer la logique de traitement des tâches, notamment l'ajout, la modification, et la suppression de tâches.

  • Développement du front-end : La partie front-end a été réalisée en HTML, CSS, et JavaScript pour offrir une interface fluide et réactive permettant aux utilisateurs d'interagir avec les tâches de manière intuitive.

Ce projet a permis de créer une première version fonctionnelle de l'application, bien qu'il y ait encore des améliorations possibles à apporter, notamment sur l'optimisation des requêtes et l'ajout de nouvelles fonctionnalités comme les notifications et les rappels.

Réalisations

Conception de la base de données : Création d'une base de données avec plusieurs tables pour gérer les utilisateurs et les tâches.

Interface utilisateur : Conception d'une interface permettant aux utilisateurs d'ajouter et de modifier des tâches facilement. La plateforme propose également un système de filtres pour voir les tâches par statut (en cours, terminées, etc.).

Technologies utilisées :

  • HTML/CSS pour la structure et la mise en forme.
  • JavaScript pour les interactions dynamiques avec l'utilisateur.
  • PHP pour la gestion du back-end et des interactions avec la base de données MySQL.
  • MySQL pour la gestion de la base de données relationnelle.

Fonctionnalités développées :

  • Formulaire pour ajouter de nouvelles tâches.

  • Page de gestion des tâches avec un tableau interactif.

  • Fonction de mise à jour et de suppression des tâches.

Compétences Utilisées

CSS pour le design et la mise en forme responsive
JavaScript pour les interactions dynamiques
PHP pour le développement back-end
